# 課金同意導線 設計（課金実装時に着手）

作成日: 2026-06-05
状態: **設計のみ**（現在、決済機能は未実装＝有料会員は「予定」。決済実装とセットで着手）

## 目的

現状は「利用＝同意とみなす」方式（免責・プライバシー）。
有料課金を始める場合、**課金/会員登録時に、規約・免責・非助言性への明示的な同意**を取る導線が必要。
これは(1)消費者法・特商法の最終確認画面規制への対応、(2)金商法 B案の「投資助言ではない」ことへのユーザー認識の確保、の両面で重要。

## 同意導線の要件

### A. 会員登録／課金フローに同意チェックを設置
- 決済確定ボタンの**直前**に、次の同意を明示的に取得（チェックボックス、デフォルト OFF）：
  - ☐ [利用規約](./02-利用規約ドラフト.md)に同意します
  - ☐ [プライバシーポリシー・免責事項](#)に同意します
  - ☐ 本サービスは**投資助言ではなく**、投資の最終判断は自己責任で行うことを理解しました
- 3つ（または統合した1つ）すべてにチェックしないと決済ボタンを押せない（disabled）。
- リンクは別タブ/モーダルで全文を読める状態にする。

### B. 最終確認画面（特商法）
- 決済前に、**金額・課金サイクル・解約条件・自動更新の有無**を明示（[特商法表記](./03-特定商取引法に基づく表記ドラフト.md)と整合）。
- 「定期課金である」「いつ・いくら課金されるか」「解約方法」を分かりやすく表示。

### C. 同意の記録
- 同意日時・同意した規約バージョンを Firestore に保存（例：`users/{uid}/data/consent` ＝ `{ termsVersion, agreedAt, agreedItems }`）。
- 規約改定時に再同意が必要なら、バージョン差分で検知して再表示。

## 実装の当たり（既存資産との接続）

- 既存の会員制御は `useCommunityAccess` / `community_members`（管理者がメール登録）。
  - 現状は**無料の招待制**。課金に移行する場合、決済事業者（Stripe 等）の Webhook で `community_members` または会員フラグを自動付与する設計に拡張。
- 同意チェック UI は、課金導線（新規の Pricing/Checkout 画面 or モーダル）に新設。
- Firestore 書き込みは、本プロジェクトの規約どおり **`firestoreRest.ts` の `restSetDoc()`** を使う（SDK の setDoc はハングする既知問題）。

## 着手の前提（ブロッカー）

1. **決済事業者の選定**（Stripe 等）と料金プラン確定（[特商法表記](./03-特定商取引法に基づく表記ドラフト.md) の【要記入】）。
2. **利用規約・特商法表記の弁護士確認完了**（同意対象の文面が固まっていること）。
3. [金商法自己点検書](./01-金商法コンプライアンス自己点検書.md) 論点A/B の結論（同意文の「投資助言ではない」表現の最終形が決まる）。
